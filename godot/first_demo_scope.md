# Минимальный скоуп первого демо

## Цель

Собрать первый standalone-стенд в Godot без full ORBAT runtime: на арене с препятствиями и `NavMesh` автоматически сражаются две команды по 10 юнитов, а параметры юнитов и их ассетов читаются из XML.

## Что входит в первый демо-срез

- Арена с плоскостью, препятствиями, рабочим `NavMesh` и стартовыми точками двух сторон.
- Автоспавн сценария `10 vs 10`.
- Один базовый тип боевого поведения: найти ближайшего врага, подойти на дистанцию, стрелять projectile-ами по кулдауну.
- Одно попадание projectile-а = мгновенная смерть цели.
- Респавн юнита по таймеру в исходной точке спавна.
- Простая обзорная камера в стиле `RTS/Total War`: панорамирование, drag, zoom, поворот вокруг вертикали, фиксированный наклон.
- XML-конфиги, через которые можно менять параметры юнитов и состав их ассетов без правки кода.

## Минимальная модель данных

Для первого демо `Unit` фиксируется как двухуровневая сущность:

- `UnitDefinition` — логический контракт юнита для runtime.
- `AssetDefinition` — описание одного ассета, на который ссылается юнит.

Минимально обязательная поддержка:

- юнит собирается из логического описания и списка ссылок на ассеты;
- ассеты читаются из XML;
- ссылки `unit -> asset` валидируются при загрузке;
- на runtime допускается один визуальный болван на весь юнит, даже если данных ассетов несколько.

Это означает, что в первом демо мы подтверждаем совместимость с двухуровневой моделью данных, но не пытаемся реализовать полный ORBAT-контур.

## Что сознательно не входит в скоуп

В первую версию намеренно не входят:

- `Formation`, `Commander`, `SALUTE`, `ROE`, `net_data`;
- full runtime для `CompositeUnit`;
- вложенные runtime-контейнеры техники, десанта и cargo;
- supply, modifiers stack, сложные статусы;
- сложная модель урона, брони и живучести ассетов;
- многоступенчатая анимационная и визуальная сборка юнита из ассетов;
- продвинутая тактика, LOS высокой точности и командные приказы.

## Практическое правило для реализации

Любая механика, которая не нужна напрямую для цикла `spawn -> move -> shoot -> kill -> respawn`, считается вне скоупа первого демо.

Любая часть ORBAT-дизайна, которая не мешает:

- загрузить юнита из XML;
- привязать к нему набор ассетов как данные;
- заспавнить его на арене;
- дать ему сражаться в сценарии `10 vs 10`,

откладывается на следующие этапы.

## Критерий готовности скоупа

Первый демо-срез считается корректно зафиксированным, если команда может дальше реализовывать стенд как ограниченный sandbox, а не как урезанную попытку собрать весь ORBAT runtime сразу.
