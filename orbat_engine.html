01 Концепция

ORBAT — игронезависимый фреймворк управления боем двух и более сторон. Логика боя полностью абстрагирована от движка и ассетов.

Принцип: одна и та же логика должна описывать и общевойсковой бой дивизии NATO, и битву при Хельмовой пади.

Разрабатывается как самостоятельная стратегическая игра-песочница на Godot Engine. Мир — Оррим, 1952 н.э.

Граф командования

BattleState └── HQ └── Commander ← роль в каждом юните └── Unit ← листовой узел ├── Order приказ = намерение ├── ROE поведение при выполнении ├── Perception что знает юнит └── Assets изменяют abilities и характеристики

03 Архитектура юнита

Юнит — листовой узел графа командования. Техника — ассеты. Юнитом является экипаж.

Любой юнит формально обладает всеми четырьмя базовыми действиями. «Тип» юнита — читаемое обозначение конкретной конфигурации коэффициентов и характеристик. Логика боя тип не читает.

ДействиеОписаниеПример
MOVEПеремещается в пространствеПехота, танки, кавалерия
SEEОбнаруживает объекты в радиусеРазведка, наблюдатель
SHOOTАтакует прямым огнём (LOS)Стрелки, танки, лучники
BLASTАтакует зону без LOSАртиллерия, катапульта, авиация

Profile — описательный слой

Для UI, рендера и звука. Алгоритм боя profile не читает.

unit.profile { label, era, domain, side }
Что делает юнита кавалерией — не поле type = cavalry, а конкретная конфигурация: высокий base.move, средний base.shoot, нулевой base.blast, плюс attached ассет «лошадь» с ability_modifiers.

04 Система abilities — base + modifiers stack

Abilities — это степень возможности выполнять действие, не сила воздействия. Сила атаки, дальность, скорость — это характеристики юнита. Abilities определяют, насколько свободно юнит может эти характеристики реализовать в текущем состоянии.

Структура

unit.abilities {
  base {                    # базовые коэффициенты — не меняются
    move, see, shoot, blast  # 0.0 – 1.0 каждый
  }
  modifiers[] {             # стек модификаторов (дельты)
    source:  string         # 'suppressed' | 'damaged' | asset.id | terrain.id
    move, see, shoot, blast  # float, может быть отрицательным
    expires: int | null     # тиков до снятия, null = постоянный
  }
  effective {               # только чтение, пересчёт каждый тик
    = clamp(base + Σ modifiers, 0.0, 1.0)
  }
}

Взаимодействие с характеристиками

effective_speed  = move_speed    × abilities.effective.move
effective_vision = vision_range  × abilities.effective.see
effective_attack = attack_damage × abilities.effective.shoot

Полный контракт юнита

  id
  profile { label, era, domain, side }   # только для UI и рендера

  parent, order, report
  perception { ... }
}

05 Система приказов

Приказ = намерение. Поведение при выполнении определяет ROE, а не приказ.
ТипПараметрыСмысл
MOVEposition: Vector3Выдвинуться в точку
HOLDposition: Vector3Удерживать точку
SUPPORTtarget: unit_id | Vector3Поддержать юнит или зону

Барраж — SUPPORT с целью-зоной. Юнит с ненулевым effective.blast и соответствующим ROE сам определяет как поддерживать.

Order { L1: type, L2: target, L3: radius, L4: expire }

06 Ассеты

Ассет меняет abilities и/или характеристики юнита. Может быть источником самой способности (лошадь даёт мобильность) или ограничивать её (верховая стрельба хуже пешей).

ТипПоведениеПример
ConsumableУничтожаетсяБоеприпасы, топливо, еда
AttachedСледует за юнитомГрузовик, БТР, лошадь
StaticОстаётся на местеМост, база, укреплённая позиция
Asset.payload {
  speed_bonus, supply_amount, supply_type, armor, comms_bonus
  ability_modifiers {       # дельты к abilities юнита
    move:  float            # лошадь: +0.4
    see:   float            # бинокль: +0.2
    shoot: float            # верховая стрельба: −0.3
    blast: float
  }
}
# при уничтожении/потере ассета его модификаторы снимаются автоматически

07 Восприятие (Perception)

ИсточникЧто даёт
Себя (self)Позиция, supply, state, abilities.effective — всегда доступно
Прямое (SEE)spotted[] в vision_range, terrain рядом
Сеть (net_data)Report'ы юнитов, данные Commander'а
spotted_object {
  position, type            # unit | asset | unknown
  classification            # vehicle | soldier | structure | ...  (перцептивный слой)
  confidence, side, side_confidence
}
classification — перцептивный слой: как юнит выглядит для наблюдателя. Не пересекается с profile и не используется алгоритмом боя напрямую.

Три режима работы

РежимУсловиеИсточник решений
Connectedcomms = onlineПриказы + ROE + net_data
Degradedcomms = degradedПоследний приказ + ROE + частичный net_data
Autonomouscomms = offlineТолько ROE + личное восприятие
Autonomous — инициатива снизу. Именно здесь партизан решает выдвинуться на помощь без приказа.

09 World Data и ZoneMap

World Data — объективная

Один, общий, принадлежит движку (NavigationServer3D). Terrain может применять ability_modifiers аналогично ассетам.

ТипПроходимостьВлияние
openground, airНет модификаторов
roadground, airБонус к move_speed
urbanground, airROE → urban_combat
fortifiedground, airСнижение входящего урона
waterair, seaНепроходимо для ground

ZoneMap — субъективная

Оценка штаба. Своя у каждой стороны. Два противника видят разное.

Green

Контролируем или безопасно

Yellow

Неопределённость, возможен контакт

Red

Вражеский контроль или бой

Любой contact report           → green → yellow
Подтверждённый, confidence>0.7  → yellow → red
Верификация чистая              → red → yellow → green
spotted[] всегда перекрывает zone_cache. Вижу сам — доверяю глазам, не карте.

10 ROE — Правила применения силы

Каскадный слой: HQ → Commander → Unit. Personality модифицирует веса.

stance не является частью приказа.

Personality

ПараметрДиапазонОписание
risk_tolerance0.0–1.0Готовность действовать при угрозе
loyalty0.0–1.0Реакция на состояние союзников
initiative0.0–1.0Автономия при offline comms

Drives (пересчёт каждый тик)

can_execute = f(
  path_exists, supply > threshold, target_reachable,
  min_ability(required) > roe.ability_threshold
)
perceived    = threat_raw × (1 − risk_tolerance)
social       = priority_delta × loyalty
search_drive = priority_delta × initiative × (1 − best_asset_score)

ability_threshold — порог выполнимости

Параметр ROE, устанавливается Commander'ом. Ниже порога — приказ считается невыполнимым.

roe.ability_threshold { move: 0.2, shoot: 0.1, see: 0.15, blast: 0.1 }

Дерево решений

0
Нет приказа + comms offline
initiative > 0.7 and priority_delta > 0.6 → AUTO_ORDER  |  HOLD
1
Можем выполнить?
can_execute > 0.6 and perceived_threat < 0.7 → EXECUTE
2
Почему не можем?
supply < threshold → SEEK_ASSET
path blocked → FIND_ALT_PATH
effective_ability < threshold → HOLD | SEEK_ASSET
search_drive > 0.3 → MOVE(search_area)
3
Угроза перевешивает?
perceived > 0.7 and can_execute > 0.5 → ENGAGE
perceived > 0.7 and can_execute < 0.5 → FALLBACK
4
Появилось что-то важнее?
social_delta > current_order.weight → SUPPORT/MOVE
5
Comms offline
initiative × can_execute > 0.5 → автономно (шаги 1–4)  |  HOLD

11 Commander

Роль, не тип сущности. При гибели — следующий по rank принимает командование автоматически.

Уничтожение командиров — легитимная тактика. Система деградирует органически.
CommanderRole { is_commander, rank, group_id, baseline_threat, roe_overrides }
next = max(rank) среди unit.state != dead в group_id

combat_power — обновлённый агрегат

unit_power(u) = u.hp × max(effective.move, effective.see, effective.shoot, effective.blast)
combat_power  = avg(unit_power) по group.units

Цикл (каждые 30 тиков)

1
aggregate(spotted[]) → update(group_state)
2
combat_power < fallback_threshold → FALLBACK всей группе
3
unit.asset_score < 0.3 and assets доступны → назначить ассет
4
known_enemies изменились → пересчитать приказы
5
send_report(parent)

12 Net Data

⏱ Задержка
К моменту передачи позиции — враг уже не там.

qualification = base_skill × unit.hp × abilities.effective.see × (1−threat_level) × logistics.supply
