01 Концепция

ORBAT — игронезависимый фреймворк управления боем двух и более сторон. Логика боя полностью абстрагирована от движка и ассетов.

Принцип: одна и та же логика должна описывать и общевойсковой бой дивизии NATO, и битву при Хельмовой пади.

Самостоятельная стратегическая игра-песочница на Godot Engine. Мир — Оррим, эпоха 1952 н.э.

BattleState
└── Formation         ← оперативный уровень (GLOBAL)
     └── Commander     ← роль в каждом юните
          └── Unit / CompositeUnit  ← тактический уровень (LOCAL)
               ├── Order      намерение + опц. L5
               ├── ROE
               ├── Perception SALUTE модель
               └── Assets     inventory / container / mutable

02 Масштабы симуляции

GLOBAL
Formation
Свёрнутая симуляция. Агрегированные abilities, фронт, логистика.
LOCAL
Unit / CompositeUnit
Полная симуляция в реальном времени.
PERSONAL
Перспектива игрока
LOCAL с фокусом внутри юнита. Отдельной боевой логики нет.
Order сверху → Objective снизу. Трансляция GLOBAL→LOCAL через Net Data с реалистичной задержкой.

03 Formation

Организационная единица оперативного уровня. Не CompositeUnit. echelon — произвольный дескриптор (squad, platoon, battalion...), не жёсткий тип.

Formation {
  id, designation, echelon, side
  command_graph { hq_unit_id, officer_list[], sub_formations[] }
  aggregate { strength, combat_power, supply, abilities{move,see,shoot,blast} }
  perception_summary { known_contacts[], zone_map_slice, confidence, last_updated }
  sync_threshold: float     # порог немедленного обновления
}

05 Архитектура юнита

Техника — ассеты-контейнеры. Юнитом является экипаж. Тип (пехота, танк...) — в profile, для UI, не для логики.

MOVE
Перемещение. Пехота, танки, кавалерия.
SEE
Обнаружение в радиусе. Разведка, БПЛА.
SHOOT
Атака прямым огнём (LOS).
BLAST
Атака зоны без LOS. Артиллерия.
abilities = clamp(base_abilities + sum(modifiers), 0.0, 1.0)  # каждый тик

unit.comms_channels {
  direct_comms {
    voice_range: float    # ~10–30 м, работает при offline
    radio_short: float    # ~200–500 м, требует comms != offline
  }
  comms_quality: float    # 0.0–1.0, для net_data pipeline
}

06 Composite Unit

id = f(container.id). БМП идентифицируется по корпусу, не по составу экипажа.

08 Восприятие: SALUTE модель

Репорт юнита структурирован вокруг военного стандарта SALUTE. Каждое поле имеет собственную достоверность — информация собирается неравномерно.

S
Size
Численность и масштаб группы. count_estimate, scale (squad/platoon/...)
A
Activity
Что делает прямо сейчас. stationary / moving / engaging / deploying / retreating...
L
Location
Позиция и её достоверность. Проще всего для обнаружения.
U
Unit
Тип и предполагаемая принадлежность. classification, unit_id_estimate.
T
Time
Когда наблюдалось. Совпадает с report.timestamp.
E
Equipment
Оснащение. Требует высокого abilities.see и близости к контакту.

Структура spotted_contact

spotted_contact {
  contact_id        # уникальный ID, присваивается первым обнаружившим
  origin_unit, last_updated

  salute {
    S { count_estimate: int|null, scale: enum, confidence: float }
    A { activity_type: enum, direction: Vector3|null, confidence: float }
    L { position: Vector3, position_confidence: float }
    U { classification: string, unit_id_estimate: string|null, confidence: float }
    T: timestamp
    E { observed_assets: [], weapon_estimate: string|null, confidence: float }
  }
  stale: bool     # помечается — не удаляется
  expires: int    # тиков до пометки stale
}

Per-field confidence: пример наблюдения в условиях неопределённости

Пехотинец наблюдает колонну в тумане войны:

L
0.90вот они — позиция точна
A
0.80двигаются на север — движение заметно
S
0.60человек десять? — оценка грубая
U
0.30пехота или разведка — неясно
E
0.10что-то несут — не разобрать

Формулы per-field confidence

qualification = base_skill * unit.hp * abilities.see * (1-threat) * logistics.supply

L.position_confidence = qualification * 1.0     # самое доступное
A.confidence          = qualification * 0.9     # движение заметно хорошо
S.confidence          = qualification * 0.8
U.confidence          = qualification * recognition_factor
E.confidence          = qualification * proximity_factor   # требует близости

# abilities.see → 0: report генерируется, все confidence → 0
# 'ничего не вижу' — тоже разведывательная информация

09 Каналы коммуникации

Трёхуровневая модель. Уровни не заменяют, а дополняют друг друга.

voice_range
📏 10–30 м
⚡ Мгновенно
✓ Работает при offline

Крик, сигналы. Даже при полном глушении связи.

radio_short
📏 200–500 м
⚡ Мгновенно
⚠ comms != offline

Ближняя рация отделения/взвода.

net_data
📏 Неограниченно
🕐 Latency + degradation
⚠ radio / physical carrier

Штатная сеть управления. Может деградировать и прерываться.

direct_comms_radius: мгновенная синхронизация

В пределах voice_range или radio_short все союзные юниты мгновенно получают contact_id и актуальный SALUTE-репорт. Достоверность = максимальная среди юнитов в радиусе, по каждому полю независимо.

# Снайпер с высоким abilities.see в радиусе:
contact.salute.E.confidence = max(u.salute.E.confidence for u in radius_group)
# Немедленно делится зрением с отделением — без Net Data pipeline

# На GLOBAL прямой коммуникации нет — только net_data
При comms = offline: voice_range работает, radio_short — нет. Глушение связи изолирует по рации, но не лишает юнитов голосовой координации вплотную.

10 Управление контактами

contact_id: гибридный подход

# Первое обнаружение:
contact_id = generate()
contact.origin_unit = unit_id
# Синхронизация внутри direct_comms_radius — мгновенно
# Распространение за пределы радиуса — через net_data с задержкой
Деконфликтация — задача штаба (Commander), не юнитов. Юниты создают contact_id независимо. Commander объединяет по proximity.

11 Четыре базовых приказа

Приказ = намерение. SPOT генерирует полный SALUTE-репорт. Пассивный contact может быть неполным.
MOVE
Выдвинуться в точку.
HOLD
Удерживать точку.
SUPPORT
Поддержать юнит или зону. Барраж = SUPPORT с зоной.
SPOT
Разведать без перемещения. Полный SALUTE → spot_result (priority 1). abilities.see=0 — выполним, confidence→0.

12 Logistics directive (L5)

Опциональный параметр приказа. Без него ROE вычисляет логистику автоматически. С ним — Commander управляет явно (только Survival прерывает).

L5: { action: MOUNT|DISMOUNT|LOAD|UNLOAD|DEPLOY|STOW,
      asset_id, timing: before|on_arrival|after }

{ L1: SPOT, L2: zone_Q, L5: { action: MOUNT, asset_id: uav_7, timing: before } }

15 ROE — Правила применения силы

Каскад HQ → Commander → Unit. Работает одинаково на LOCAL и GLOBAL.
0
Нет приказа + offline
initiative > 0.7 and priority_delta > 0.6 → AUTO_ORDER else → HOLD
1
Есть L5 директива?
выполнить в timing # Survival прерывает
2
can_execute > 0.6 and perceived < 0.7
→ EXECUTE
3
Почему не можем?
supply < threshold / path blocked / ability < threshold container_crew < threshold / mutable stowed / search_drive > 0.3
4
Угроза
perceived > 0.7 → ENGAGE / FALLBACK
5
Что-то важнее?
social_delta > order.weight → SUPPORT/MOVE
6
Offline
initiative * can_execute > 0.5 → автономно else → HOLD

16 Commander

Роль в каждом юните. При гибели — следующий по rank. На LOCAL: HQ Formation = наибольший по иерархии доступный Commander.

  roe_overrides {

    require_mounted:         bool

  }
}
